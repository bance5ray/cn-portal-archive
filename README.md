# cn-portal-archive

本仓库用于归档和发布多个独立的 HTML 页面，内容涵盖各类信息呈现与交互展示。

## 项目简介

cn-portal-archive 是一个静态页面归档项目，通过 Git 版本管理保存 HTML 文件的迭代历史。仓库中的页面均以独立文件形式存放，不依赖后端服务，可直接在浏览器中打开或通过静态托管服务访问。

该仓库适用于以下场景：
- 记录和保存特定时间点的页面快照
- 提供离线可查看的 HTML 资源
- 归档多个互不关联的独立页面

## 目录结构

```
.
├── pages/               # 存放所有归档 HTML 页面
│   ├── example.html     # 示例页面
│   └── ...
├── assets/              # 公共资源文件（CSS、JS、图片等）
│   ├── styles/
│   ├── scripts/
│   └── images/
└── README.md            # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立的归档页面，文件名应具有描述性。
- `assets/`：存放页面共用的样式、脚本及媒体文件，便于复用和维护。

## 页面归档说明

- 所有页面均使用标准 HTML 编写，不依赖外部 API 或数据库。
- 页面间相互独立，无交叉引用，可单独部署或查看。
- 归档时建议保留页面原有的结构、样式与基本交互功能。
- 如需更新页面内容，请通过提交新的 commit 或替换文件实现。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来新增、更新或修正归档页面。
- 提交新页面时，请将其放入 `pages/` 目录，并确保文件命名清晰。
- 若页面需要额外资源，请放置于 `assets/` 对应子目录中。
- 请勿在页面中包含任何动态加载的外部资源（如外链脚本），以保证离线可用性。
